# Vue Setup

Vue 3 supports KarbitUI in two modes:

- raw custom elements with `kb-*`
- official Vue wrappers from `@wearesyntesa/karbit-ui/vue`

Use the wrappers for application forms and use the raw custom element path when you want direct web-component access.

## Installation

```bash
pnpm add @wearesyntesa/karbit-ui vue
```

## Official Vue wrappers

The Vue wrappers remove the most common pain points in forms:

- no `isCustomElement` config for wrapped controls
- no manual `CustomEvent` casting
- no `.prop` binding for `Select` options
- named `v-model` bindings that match the underlying API

```vue title="src/App.vue"
<script setup lang="ts">
import { Checkbox, Input, Select, Switch, Textarea } from '@wearesyntesa/karbit-ui/vue';
import { ref } from 'vue';

const name = ref('');
const role = ref('');
const notes = ref('');
const accepted = ref(false);
const notifications = ref(false);

const roleOptions = [
  { value: 'admin', label: 'Admin' },
  { value: 'editor', label: 'Editor' },
  { value: 'viewer', label: 'Viewer' },
];
</script>

<template>
  <div class="space-y-4">
    <Input v-model:value="name" placeholder="Name" />
    <Select v-model:value="role" :options="roleOptions" placeholder="Role" />
    <Textarea v-model:value="notes" placeholder="Notes" />
    <Checkbox v-model:checked="accepted">Accept terms</Checkbox>
    <Switch v-model:checked="notifications">Notifications</Switch>
  </div>
</template>
```

## Raw custom element path

The raw path is still supported and is the right choice when you want the direct web-component API.

### Configure Vue

Tell Vue to treat `kb-*` tags as custom elements so it doesn't try to resolve them as Vue components:

```ts title="vite.config.ts"
import { defineConfig } from "vite";
import vue from "@vitejs/plugin-vue";

export default defineConfig({
  plugins: [
    vue({
      template: {
        compilerOptions: {
          isCustomElement: (tag) => tag.startsWith("kb-"),
        },
      },
    }),
  ],
});
```

### Register custom elements

```ts title="src/main.ts"
import "@wearesyntesa/karbit-ui/define";

import { createApp } from "vue";
import App from "./App.vue";

createApp(App).mount("#app");
```

### Usage example

```vue title="src/App.vue"
<script setup lang="ts">
function handleChange(e: CustomEvent) {
  console.log("Value:", e.detail.value);
}
</script>

<template>
  <kb-card p="6">
    <kb-input
      placeholder="Enter your name"
      @kb-change="handleChange"
    />
    <kb-button variant="solid" color-scheme="blue" mt="4">
      Submit
    </kb-button>
  </kb-card>
</template>
```

### Event handling

KarbitUI components emit `CustomEvent`s. In Vue, listen with `@event-name`:

```vue
<template>
  <!-- Kebab-case event names -->
  <kb-select @kb-change="onSelect" />
  <kb-checkbox @kb-change="onToggle" />
  <kb-tabs @kb-change="onTabChange" />
</template>
```

### Attribute vs. property binding

Vue binds attributes by default. For complex data (arrays, objects), use `.prop` modifier or `v-bind`:

```vue
<template>
  <!-- String/boolean attributes work directly -->
  <kb-button variant="solid" disabled>Disabled</kb-button>

  <!-- Complex data needs property binding -->
  <kb-table :columns.prop="columns" :rows.prop="rows" />
  <kb-tree-view :items.prop="treeItems" />
  <kb-select :options.prop="options" />
</template>
```

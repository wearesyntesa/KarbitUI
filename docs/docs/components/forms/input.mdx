## Props

| Property | Attribute | Type | Default | Description |
|---|---|---|---|---|
| `inputId` | `input-id` | `string` | — | Forwarded `id` applied to the native `<input>` element. Allows external `<label for="...">` to work. |
| `variant` | `variant` | `FormVariant` | `'outline'` | Form input visual variant. |
| `size` | `size` | `ComponentSize` | `'md'` | Input size controlling padding, font size, and icon sizing. |
| `placeholder` | `placeholder` | `string` | — | Placeholder text shown when the input is empty. |
| `value` | `value` | `string` | `""` | Current input value. Two-way bindable. |
| `type` | `type` | `InputType` | `'text'` | HTML input type attribute (e.g. `'text'`, `'password'`, `'email'`). |
| `name` | `name` | `string` | — | Form field name, used in form submissions. |
| `maxLength` | `max-length` | `number` | — | Maximum character count. When set, enforces length via the native `maxlength` attribute. |
| `disabled` | `disabled` | `boolean` | `false` | Disable interaction and apply dimmed styling. |
| `invalid` | `invalid` | `boolean` | `false` | Mark the input as invalid with error border styling. |
| `readonly` | `readonly` | `boolean` | `false` | Make the input read-only (focusable but not editable). |
| `required` | `required` | `boolean` | `false` | Mark the input as required for form validation. |
| `clearable` | `clearable` | `boolean` | `false` | Show a clear button when the input has a value. |
| `loading` | `loading` | `boolean` | `false` | Show a loading spinner in place of the trailing icon. |

## Slots

| Name | Description |
|---|---|
| `addon-left` | Attached text/element flush to the left edge (e.g. "https://"). |
| `addon-right` | Attached text/element flush to the right edge (e.g. ".com"). |
| `icon-left` | Leading icon element inside the input. |
| `icon-right` | Trailing icon element inside the input. |

## Events

| Event | Detail | Description |
|---|---|---|
| `kb-input` | `{ value: string }` | Value changed on input. Detail: `{ value: string }`. |
| `kb-change` | `KbChangeDetail` — discriminated union, narrow via `detail.source` | Value committed on change. Detail: `{ value: string }`. |
| `kb-focus` | `{ source?: string }` or none | Input focused. |
| `kb-blur` | `{ source?: string }` or none | Input blurred. |
| `kb-clear` | `{ source?: string }` or none | Clear button clicked (when clearable). |
